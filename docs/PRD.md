# PRD: My AI Usage Audit

## 1. Project Overview

본 프로젝트는 개발자들이 Claude Code를 사용할 때의 습관을 진단하고, '무지성 사용'을 방지하며 토큰 효율 및 에이전트 활용 능력을 높이기 위한 인터랙티브 웹 서비스이다.

## 2. Target Audience

- Claude Code를 유료로 결제하여 사용 중인 현업 개발자.
- AI가 생성하는 코드의 품질과 비용(토큰) 최적화에 관심이 있는 사용자.

## 3. Core Values

- **Awareness:** 본인의 비효율적인 AI 사용 습관 인지.
- **Mastery:** `/clear`, `/compact`, `CLAUDE.md` 등 고급 기능 학습.
- **Efficiency:** 전략적 세션 관리를 통한 비용 절감 및 속도 향상.

## 4. Key Features

### 4.1. Interactive Audit Quiz (7문항, 카테고리 기반)

선택지에 따른 실시간 'Token Gauge' 및 'Terminal Log' 애니메이션 피드백과 함께, 7개 카테고리로 사용자의 Claude Code 활용 수준을 진단한다.

#### 진단 카테고리 및 질문 설계

| # | 카테고리 | 시나리오 예시 | 진단 포인트 |
|---|---------|-------------|------------|
| 1 | **Session 관리** | 컨텍스트가 커져서 응답이 느려졌다. 어떻게 하는가? | `/compact`, `/clear` 활용 여부 |
| 2 | **프롬프트 품질** | 새 기능을 구현하려 한다. 첫 메시지로 뭘 보내는가? | 요구사항 구조화 vs "해줘" |
| 3 | **에러 대응** | AI가 같은 에러를 3번째 반복한다. 어떻게 하는가? | Interrupt → 맥락 리셋 vs 무한 재시도 |
| 4 | **코드 검증** | AI가 생성한 200줄 코드가 돌아간다. 다음 행동은? | diff 리뷰/테스트 vs 바로 커밋 |
| 5 | **메모리 활용** | 프로젝트 컨벤션을 매번 설명하고 있다. 어떻게 개선하는가? | `CLAUDE.md`, memory 활용 |
| 6 | **도구 활용** | 여러 파일을 동시에 수정해야 한다. 어떤 접근을 취하는가? | Agent, multi-edit 등 고급 기능 활용 |
| 7 | **워크플로우 설계** | 대규모 리팩토링을 해야 한다. 어떻게 시작하는가? | Plan 모드 → 단계별 분리 vs 한 번에 전부 시키기 |

각 질문은 3~4개 선택지를 제공하며, 선택지마다 **score**(실력 점수)와 **tokenImpact**(효율 점수)를 분리 채점하여 2축 평가를 수행한다.

### 4.2. Result & Character System (2축 매트릭스)

실력(score)과 효율(tokenImpact) 두 축으로 캐릭터를 부여한다.

```
         효율 높음
            │
   Prompt   │   Token
   Architect│   Alchemist
            │
  ──────────┼──────────── 실력
            │
   Hopeful  │   Brute Force
   Beginner │   Coder
            │
         효율 낮음
```

| 캐릭터 | 조건 | 한 줄 설명 |
|--------|------|-----------|
| **Token Alchemist** | 실력 높 + 효율 높 | 최소 토큰으로 최대 결과를 뽑아내는 마법사 |
| **Prompt Architect** | 실력 높 + 효율 보통 | 잘 쓰지만 아직 최적화 여지가 있는 설계자 |
| **Brute Force Coder** | 실력 보통 + 효율 낮 | 결과는 내지만 토큰을 불태우는 전사 |
| **Hopeful Beginner** | 실력 낮 + 효율 낮 | 가능성은 무한한 AI 코딩 입문자 |
| **Copy-Paste Gambler** | 실력 낮 + 효율 높 | 검증 없이 빠르게 복붙하는 도박사 |

캐릭터별 특징 요약 및 강점/약점 분석을 제공한다.

### 4.3. Actionable Insights (카테고리 기반 맞춤 팁)

각 질문에서 최적이 아닌 선택을 한 카테고리에 대해 맞춤 팁을 노출한다.

- 낮은 점수 카테고리별 구체적 개선 가이드 (예: Session 관리가 약하면 → `/compact` 사용법 안내)
- 바로 복사해서 사용할 수 있는 `CLAUDE.md` 템플릿 및 워크플로우 스니펫 제공.

### 4.4. Social Sharing

- 진단 결과(캐릭터 카드) 이미지 저장 및 SNS 공유 기능.

## 5. Success Metrics

| 지표 | 측정 방법 | 목표 |
|------|----------|------|
| 퀴즈 완료율 | 시작 대비 결과 페이지 도달 | > 70% |
| 공유 클릭율 | 결과 페이지 대비 공유 버튼 클릭 | > 20% |
| 팁 복사율 | 코드 스니펫 복사 버튼 클릭 수 | > 30% |
| 커뮤니티 반응 | GitHub Star, GeekNews 등 개발자 커뮤니티 반응 | 긍정적 |
